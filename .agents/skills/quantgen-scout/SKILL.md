# Quantgen Scout

## Procedure

1. Define the design or claim being checked.
2. Read `references/packages.md` for the local comparison map.
3. Prefer primary sources, official docs, papers, or package vignettes.
4. Record what hsquared should learn, avoid, or defer.
5. Update `docs/dev-log/scout/` or the relevant design note when the lesson
   should persist.

## Output

Return:

- question scouted;
- sources checked;
- relevant lesson;
- hsquared action;
- claim wording risk, if any.
