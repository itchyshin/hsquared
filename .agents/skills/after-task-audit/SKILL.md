# After-Task Audit

## Procedure

Create a report under `docs/dev-log/after-task/` with:

- task goal;
- active lenses and spawned agents;
- files changed;
- checks run and exact outcomes;
- public claim audit;
- tests of the tests;
- coordination notes;
- what did not go smoothly;
- known limitations;
- next actions.

Update `docs/dev-log/check-log.md`, `docs/dev-log/coordination-board.md`, and
any relevant capability or validation status row.

## Rule

A task is not done until uncertainty and next actions are explicit.
