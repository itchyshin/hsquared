# V0.1 Contract

The first implementation target is a univariate Gaussian animal model.

## R Target

```r
fit <- hsquared(
  y ~ sex + age + animal(1 | id, pedigree = ped),
  data = dat,
  family = gaussian(),
  REML = TRUE
)
```

The same contract can use the pedigree stored in an `hs_data()` bundle:

```r
bundle <- hs_data(phenotypes = dat, pedigree = ped)

fit <- hsquared(
  y ~ sex + age + animal(1 | id),
  data = bundle,
  family = gaussian(),
  REML = TRUE
)
```

## Julia Target

```julia
pedigree = normalize_pedigree(id, sire, dam)
Ainv = pedigree_inverse(pedigree)
spec = animal_model_spec(y, X, Z, Ainv; ids = ids, method = :REML)
fit = fit_animal_model(spec)
```

## R Responsibilities

- Parse the model formula.
- Validate data and pedigree columns.
- Accept either a plain phenotype data frame or an `hs_data()` bundle for the
  v0.1 phenotype/pedigree path.
- Build the fixed-effect design matrix `X`.
- Build sparse incidence matrix `Z` for observed animal IDs.
- Normalize pedigree order and parent indices for Julia `Ainv` construction.
- Marshal a normalized model specification to `HSquared.jl`.
- Return an S3 `hsquared_fit` object.
- Provide R-style summaries and extractors.

## Julia Responsibilities

- Validate the R-normalized IDs and pedigree payload.
- Build sparse Ainv directly.
- Validate `animal_model_spec(y, X, Z, Ainv; ids, method)`.
- Fit the Gaussian animal model by ML or REML.
- Return a compact result object and diagnostics.

## Current R Status

The R package currently parses the formula and builds an internal bridge
payload with `y`, `X`, sparse `Z`, method, family, encoded IDs, normalized
pedigree records, and parent indices. It also exposes `model_spec()` so users
and developers can preview that contract without fitting a model. The parser
can read model variables from a plain data frame or from the `phenotypes`
component of an `hs_data()` bundle, with formula terms such as
`pedigree = pedigree` resolved from the bundle. The R package also defines the
first internal `hsquared_fit` object and extractor methods for Julia result
fields. When `data` is an `hs_data()` bundle with a pedigree component,
`animal(1 | id)` uses that bundle pedigree by default.

There is now a local-only experimental JuliaCall path for the tiny payload when
a sibling `HSquared.jl` checkout is available. The default `hsquared()` call
still validates and stops, while
`control = hs_control(engine = "julia")` opts into the experimental bridge and
can return an internal `hsquared_fit` for tiny examples. General fitted
animal-model support remains blocked on validation-canon evidence and a stable
production bridge.

The opt-in bridge can also run a supplied-variance Henderson MME target:

```r
hs_control(
  engine = "julia",
  engine_control = list(
    target = "henderson_mme",
    variance_components = c(sigma_a2 = 1.2, sigma_e2 = 0.8)
  )
)
```

This is for tiny validation examples. It solves fixed effects and EBVs/BLUPs at
supplied variance components and can attach dense validation-path PEV and
reliability when the sibling Julia checkout exposes applicable extractors. It
does not estimate variance components.

## V0.1 Promotion Predicate

The default `hsquared()` call stays `engine = "validate"` (parse, build the
bridge payload, then stop). This predicate is **binding**: no slice may flip the
default to a real fit, or promote the `simple Gaussian animal model` /
`univariate Gaussian animal model` capability rows from `planned`/`partial` to
`covered`, until every item below holds and Rose records the audit. Internal
self-agreement (R reference solve vs Julia at supplied variances),
start-independence, and dense-vs-sparse-vs-pure-R agreement prove optimizer
reproducibility on the same objective — they are **not** known-truth evidence
and do not satisfy this predicate.

1. Twin `HSquared.jl` `validation_status()` row `V1-MRODE-FIT` reads `covered`
   or `covered_external`: a fitted animal-model example whose **estimated**
   (REML) variance components, heritability, and EBVs match a named, citable
   published source — not hsquared's own reference solve, and not a
   supplied-variance BLUP example relabelled as estimation — at declared
   tolerances.
2. Twin row `V1-COMPARATORS` reads `covered`/`covered_external`: an external
   fitted-output comparator (e.g. sommer, an ASReml-family package, or a cleanly
   converging pedigreemm design) **agrees** on the estimated variance
   components, h2, and EBVs within a declared band. The existing
   "at-least-as-good by REML log-likelihood" pedigreemm check is a one-sided
   lower bound and does **not** satisfy this item.
3. At least one estimator row (`V1-SPARSE-REML-OPT` or `V1-AI-REML`) reads
   `covered` with known-truth recovery evidence: a replicated DGP study showing
   near-unbiased recovery of the generating variance components and EBV accuracy
   tracking the simulated true breeding values, against stated thresholds. The
   study must, at minimum, (i) use a fixed seed and a stated number of
   replicates, (ii) report the relative bias of `sigma_a2`, `sigma_e2`, and h2
   with its Monte Carlo standard error, with `0` inside the bias ± 2·MCSE band
   (or `|relative bias|` below a declared cap), and (iii) report the mean
   `cor(EBV, true BV)` above a declared floor. The numeric thresholds are
   maintainer-owned, but this form is non-optional: "near-unbiased" may not be
   decided ad hoc from a handful of replicates.
4. A boundary / identifiability diagnostic: the promoted fit must distinguish an
   interior optimum from a `sigma_a2 = 0` (or `sigma_e2 = 0`) boundary solution
   and surface that through `fit_diagnostics()`, and `summary()` must not present
   a boundary h2 as an ordinary interior estimate. The R-side **surfacing** is
   implemented: `fit_diagnostics()` reports an `at_boundary` flag derived from the
   returned variance components, and `summary()` prints a boundary note when a
   component is at/near zero. The **engine-stability** half — the optimizer
   returning a boundary-consistent, finite, documented result as `h2 -> 0/1`
   (not a crash, a negative variance, or a silently-pinned estimate) — remains
   twin (`HSquared.jl`) work; the twin already exercises a `sigma_a2 = 0` boundary
   fixture to extend.

Until the predicate holds, the opt-in `engine = "julia"` paths
(`fit_animal_model`, `henderson_mme`, `sparse_reml`, `ai_reml`) remain
experimental, gated, and explicitly not the default.

Open decisions owned by the coordinator/maintainer (must not be chosen silently
by an autonomous agent): which published estimated-VC source anchors
`V1-MRODE-FIT`; which external comparator(s) count for `V1-COMPARATORS` and the
agreement band; and the DGP-recovery pass thresholds.

## V0.1 Outputs

- Variance components.
- Heritability.
- Fixed effects.
- EBVs/BLUPs.
- Dense validation-path PEV and reliability for tiny opt-in Julia bridge
  examples when the sibling Julia checkout exposes those extractors.
- Accuracy as square-root reliability when reliability estimates are present.
- Log-likelihood.
- Fitted values and residuals when fitted-value predictions and response values
  are available.
- Convergence diagnostics.
- Minimal fitted metadata.

## Uncertainty Scope

Validated standard errors and confidence intervals for the variance components
and heritability are **out of v0.1 scope**. `summary()` and the extractors
report point estimates only and must not print or imply an interval for variance
components or h2. PEV and reliability are reported on the dense validation path
only and are labelled as such; `accuracy()` is the square root of reliability,
not an independent accuracy estimate. A validated SE/interval (for example from
the AI-REML information matrix, checked against a known SE) is deferred to a
later milestone.

The EBV accuracy used as promotion evidence in item 3 of the predicate is the
simulation-only correlation between estimated EBVs and the known true breeding
values. It is distinct from the shipped `accuracy()`, which is the square root of
reliability — a model-internal quantity reported on the dense validation path and
not validated against realised accuracy in v0.1.

## Estimable Quantities (v0.1 identifiability)

For the v0.1 contract `y ~ fixed + animal(1 | id, pedigree = ped)`, Gaussian,
REML, with one record per animal, the estimable quantities are (standard
animal-model identifiability; Henderson 1984; Mrode 2014; Lynch & Walsh 1998):

- Additive-genetic variance `sigma_a2` and residual variance `sigma_e2` (hence
  heritability `h2 = sigma_a2 / (sigma_a2 + sigma_e2)`), identified from the
  pedigree-induced covariance among relatives (the numerator relationship matrix
  `A`). Identifiability requires informative relatedness; flat/uninformative
  pedigrees make the two components weakly separable.
- Fixed effects in the `fixed` part of the formula.
- Breeding values (EBVs/BLUPs) for animals in the pedigree.

NOT identifiable / NOT in v0.1 scope for this model (each needs structure the
v0.1 model does not contain):

- A permanent-environment / repeatability component — needs repeated records per
  individual (v0.1 is one record per animal).
- Maternal or common-environment effects — need the maternal/common grouping
  structure (Phase 2).
- Non-additive (dominance, epistasis) variance — needs the corresponding
  relationship structure (Phase 7).
- Standard errors / confidence intervals for variance components or h2 — out of
  v0.1 scope (see Uncertainty Scope).

Boundary regimes (`h2 -> 0` or `h2 -> 1`, low-information designs) are where these
estimates are least reliable; a known-truth recovery grid
(`data-raw/dgp-recovery-study.R`) shows near-unbiased recovery across the
interior (h2 = 0.2/0.4/0.6) and the expected mild upward bias plus occasional
boundary pinning near h2 = 0.1. The promotion predicate (item 4) requires the
promoted fit to surface a boundary/identifiability diagnostic through
`fit_diagnostics()` and to not present a boundary estimate as an ordinary
interior one; that diagnostic is engine (`HSquared.jl`) work and is not yet
built. The exact estimable-quantities wording and the boundary-diagnostic
contract are proposals for maintainer (Fisher/Henderson/Noether) sign-off.

## Explicit Non-Scope

The following are planned but not v0.1:

- multivariate models;
- `cov = us()`, `cov = fa()`, and `lowrank()`;
- genomic and single-step models;
- GLLVM-style non-Gaussian high-dimensional models;
- GPU paths;
- unusual inheritance systems.
