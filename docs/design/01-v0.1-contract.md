# V0.1 Contract

The first implementation target is a univariate Gaussian animal model.

## R Target

```r
fit <- hsquared(
  y ~ sex + age + animal(1 | id, pedigree = ped),
  data = dat,
  family = gaussian(),
  REML = TRUE
)
```

The same contract can use the pedigree stored in an `hs_data()` bundle:

```r
bundle <- hs_data(phenotypes = dat, pedigree = ped)

fit <- hsquared(
  y ~ sex + age + animal(1 | id),
  data = bundle,
  family = gaussian(),
  REML = TRUE
)
```

## Julia Target

```julia
pedigree = normalize_pedigree(id, sire, dam)
Ainv = pedigree_inverse(pedigree)
spec = animal_model_spec(y, X, Z, Ainv; ids = ids, method = :REML)
fit = fit_animal_model(spec)
```

## R Responsibilities

- Parse the model formula.
- Validate data and pedigree columns.
- Accept either a plain phenotype data frame or an `hs_data()` bundle for the
  v0.1 phenotype/pedigree path.
- Build the fixed-effect design matrix `X`.
- Build sparse incidence matrix `Z` for observed animal IDs.
- Normalize pedigree order and parent indices for Julia `Ainv` construction.
- Marshal a normalized model specification to `HSquared.jl`.
- Return an S3 `hsquared_fit` object.
- Provide R-style summaries and extractors.

## Julia Responsibilities

- Validate the R-normalized IDs and pedigree payload.
- Build sparse Ainv directly.
- Validate `animal_model_spec(y, X, Z, Ainv; ids, method)`.
- Fit the Gaussian animal model by ML or REML.
- Return a compact result object and diagnostics.

## Current R Status

The R package currently parses the formula and builds an internal bridge
payload with `y`, `X`, sparse `Z`, method, family, encoded IDs, normalized
pedigree records, and parent indices. It also exposes `model_spec()` so users
and developers can preview that contract without fitting a model. The parser
can read model variables from a plain data frame or from the `phenotypes`
component of an `hs_data()` bundle, with formula terms such as
`pedigree = pedigree` resolved from the bundle. The R package also defines the
first internal `hsquared_fit` object and extractor methods for Julia result
fields. When `data` is an `hs_data()` bundle with a pedigree component,
`animal(1 | id)` uses that bundle pedigree by default.

There is now a local-only experimental JuliaCall path for the tiny payload when
a sibling `HSquared.jl` checkout is available. The default `hsquared()` call
still validates and stops, while
`control = hs_control(engine = "julia")` opts into the experimental bridge and
can return an internal `hsquared_fit` for tiny examples. General fitted
animal-model support remains blocked on validation-canon evidence and a stable
production bridge.

The opt-in bridge can also run a supplied-variance Henderson MME target:

```r
hs_control(
  engine = "julia",
  engine_control = list(
    target = "henderson_mme",
    variance_components = c(sigma_a2 = 1.2, sigma_e2 = 0.8)
  )
)
```

This is for tiny validation examples. It solves fixed effects and EBVs/BLUPs at
supplied variance components and can attach dense validation-path PEV and
reliability when the sibling Julia checkout exposes applicable extractors. It
does not estimate variance components.

## V0.1 Outputs

- Variance components.
- Heritability.
- Fixed effects.
- EBVs/BLUPs.
- Dense validation-path PEV and reliability for tiny opt-in Julia bridge
  examples when the sibling Julia checkout exposes those extractors.
- Accuracy as square-root reliability when reliability estimates are present.
- Log-likelihood.
- Fitted values and residuals when fitted-value predictions and response values
  are available.
- Convergence diagnostics.
- Minimal fitted metadata.

## Explicit Non-Scope

The following are planned but not v0.1:

- multivariate models;
- `cov = us()`, `cov = fa()`, and `lowrank()`;
- genomic and single-step models;
- GLLVM-style non-Gaussian high-dimensional models;
- GPU paths;
- unusual inheritance systems.
