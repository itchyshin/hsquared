# After-Task Protocol

Every meaningful task should leave a compact report under
`docs/dev-log/after-task/`.

## Required Sections

- Task goal.
- Active lenses and spawned agents.
- Files created or changed.
- Checks run and exact outcomes.
- Public claim audit.
- Tests of the tests.
- Coordination notes.
- What did not go smoothly.
- Known limitations.
- Next actions.

## Closing Rule

A task is not done until the report says what was checked, what remains
uncertain, and what should happen next.
