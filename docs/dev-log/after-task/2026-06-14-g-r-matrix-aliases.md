## Public claim audit

Rose verdict: clean.

`G_matrix()` and `R_matrix()` are aliases only. They do not add factor-analytic
G matrices, structured covariance grammar, production multivariate fitting,
external-comparator parity, known-truth recovery, standard errors, or a
phenotypic `P_matrix()` estimand.

## Tests of the tests

The tests compare `G_matrix(fit)` directly to `genetic_covariance(fit)` and
`R_matrix(fit)` directly to `residual_covariance(fit)` on both a minimal
`hsquared_fit` and the shared Phase 4 multivariate parity fixture. Default
methods still fail for non-`hsquared_fit` objects.

## Known limitations

`P_matrix()` is intentionally not added here. Phenotypic covariance is
straightforward for the current `G + R` validation-scale model, but becomes
model-dependent once permanent environment, common environment, maternal,
genomic, or structured residual components enter the same fit.
