# After-task — LOCO marker-scan bridge (2026-06-20 s6, completes #4)

## Goal

Surface the engine's leave-one-group-out (LOCO) marker scan from R, completing
backlog #4. Handoff-8 flagged LOCO as a genuine **design** problem (genomic
per-group precisions vs a pedigree fit), not a mechanical wire-up, and required a
**live dimension probe** before any build-spec.

## Design question — resolved by a live probe

`/tmp/loco_probe.R` (run on the bridge) settled the open question. The LOCO
precision enters the `Ainv` slot of the scan's GLS covariance, and the engine
guards `size(Z, 2) == size(precision, 1)`, so:

- precisions are built from **animal-level** markers → `(n_animals × n_animals)`;
- the scan's `markers` argument is **record-level** (`Z %*% markers`);
- a **single** supplied `σ²a/σ²e` is reused from the pedigree fit (the engine LOCO
  scan takes one σ²a — per-group VC re-estimation is not in the contract).

Probe evidence: precisions `80×80`; chr1 markers match a single
`mixed_model_marker_scan` with the chr1 precision to **max|diff| = 0.0**; LOCO
differs from the whole-G scan. Build-spec: `docs/design/26-loco-gwas-bridge.md`
(IMPLEMENTED).

## Honesty

- Experimental, validation-scale, **uncalibrated** (engine gate `HSquared.jl#48`);
  no `covered` promotion.
- The LOCO relationship is **genomic** while the reused VCs are
  **pedigree**-estimated — a scale mismatch stated in `print()`/`autoplot()`/docs;
  the loco `print()` adds that the effect/SE scale is not a calibrated genomic-VC
  quantity (use ranking, not magnitudes).
- Adversarial verify (5-lens Workflow) caught **no true blocker**. Fixes applied:
  the stale `mixed`-branch print line ("LOCO … not yet surfaced") and — the real
  one — **Curie's test gap**: the live tests used a square `Z` (one record per
  animal), so animal-level ≡ record-level markers and a markers/markers_rec swap
  passed undetected. Added a non-square-Z (110 records / 80 animals, interior
  σ²a=1.12) live test that asserts the scan runs, parity vs a direct engine LOCO
  scan, and `expect_error` on the wrong wiring (record-level markers → precisions,
  rejected by the engine size guard).
